# 测试发稿工作流

## 目标
让用户提供一个素材、链接或主题，按指定 style 跑一篇测试稿并保存到微信公众号草稿箱。

## 输入
- `style_id`
- 素材 / 链接 / 主题
- 是否启用文生图

## 执行顺序
1. 按 style 渲染标题、摘要、正文
2. 规划封面图与正文图（如 style 启用）
3. 调 Tuzi 生图（如启用）
4. 上传封面图与正文图
5. 渲染公众号 HTML
6. 创建微信草稿
7. 回执草稿 `media_id`

## 默认原则
- 第一版优先保证“测试草稿闭环跑通”
- 不默认强加二维码
- 不默认强加私有引流 CTA
- 若 style 文件明确要求二维码或 CTA，再按 style 执行

## 交付给用户
- 使用的是哪个 style
- 封面图是否成功
- 正文图是否成功
- 微信草稿 `media_id`
- 建议用户重点检查：标题、文风、图文贴合度、是否像自己的号
