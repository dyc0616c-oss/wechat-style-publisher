# 风格学习工作流

## 目标
从用户提供的 5-10 篇历史公众号文章中，提炼一个可复用的 style 文件。

## 输入
- 公众号名
- 5-10 篇历史文章
- 可选：账号定位说明、目标读者说明

## 提取维度
1. 标题
   - 长度
   - 常见句式
   - 情绪强度
2. 摘要
   - 长度
   - 是信息差型、观点型还是钩子型
3. 正文结构
   - 段落数
   - 是否一句一段
   - 每段平均句数
4. 语言风格
   - 口语化程度
   - 是否犀利 / 稳健 / 故事化 / 教程化
   - 高频词与常见句式
5. 排版风格
   - 重点句是否高亮
   - 重点词是否高亮
   - 小标题是否存在
6. 结尾结构
   - 是否有引流钩子
   - 是否带二维码
   - CTA 风格
7. 图片策略
   - 封面图风格
   - 正文图风格
   - 图文插入位置

## 输出
- 生成一个 `style-x.md`
- 更新 `references/styles/account-style-map.json`
- 给用户回执“这个 style 学到了什么”

## 注意
- 不要只总结抽象形容词，要总结可执行规则
- 风格总结优先写成规则，而不是散文描述
